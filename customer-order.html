Customer Order Module

Home / Order Modules / Customer Orders

Overview

The Customer Order module manages all sales orders placed by customers, providing a complete workflow from order creation to delivery.

Key Features

Feature Description
Customer Management Create orders linked to customer records
Itemized Orders Add multiple items with variations and pricing
Status Workflow Track order progress through planned, released, cut, started, completed states
Order Handling Specialized interface for managing order fulfillment
Audit Trail Track all changes with user and timestamp information
PDF Generation Print professional order documents

Viewing Customer Orders

To browse and search through customer orders:

  1. Navigate to Orders > Customer Orders
  2. The system displays all orders in a sortable table
  3. Key columns include:
    • Order ID - Unique 5-digit identifier
    • Customer Name - Linked customer record
    • Order Name - Descriptive name
    • Wanted Delivery Date - Requested date
    • Status - Current workflow state
    • Order Total - Calculated grand total
  4. Use the search bar to find specific orders
  5. Click column headers to sort records
  6. Apply filters to narrow results
Tip: Use the "Handle" action to manage individual orders through their lifecycle.


Available Filters

  • Status - Filter by order state (planned, released, etc.)
  • Created Date - Filter by order creation date
  • Wanted Delivery Date - Filter by requested delivery date

Creating a Customer Order

To create a new customer order:

  1. Click Create Customer Order from the orders list
  2. Complete the form fields:

    Customer Information

    • Customer - Search and select from existing customers
    • Customer details auto-populate (name, phone, email)

    Order Details

    • Order Name - Descriptive name for the order
    • Wanted Delivery Date - Requested delivery date
    • Special Notes - Additional instructions

    Order Items

    • Add items using the "Add Order Item" button
    • For each item:
      • Item Name - Product description
      • Is Variation - Toggle for items with variations
      • For non-variation items:
        • Quantity
        • Price
        • Total (auto-calculated)
      • For variation items:
        • Add sub-items with "Add Variation Item"
        • Each variation has name, quantity, price, and total
  3. Click Save to create the order
Note: The grand total is automatically calculated from all items and variations.


Creating from Sample Orders

The system allows you to convert accepted sample orders into full customer orders:

  1. Navigate to Orders > Customer Orders
  2. Click Create from Sample Order (special button)
  3. Select an accepted sample order from the dropdown:
    • Only sample orders with "accepted" status are available
    • Each entry shows ID, customer name, and grand total
  4. The system auto-populates:
    • Customer information (read-only)
    • Original delivery date (read-only)
    • Order items with variations (read-only)
    • Grand total (read-only)
  5. Edit the following fields:
    • New Delivery Date - Set the actual delivery date
    • Order Items - Modify quantities/prices as needed
  6. Click Create Customer Order to complete the conversion
Automatic Updates: When converted, the sample order will be marked as "converted" and linked to the new customer order.


Editing a Customer Order

To modify an existing customer order:

  1. Locate the order in the list view
  2. Click the Edit button (pencil icon)
  3. Make changes to editable fields:
    • Order Name - Update descriptive name
    • Wanted Delivery Date - Change requested date
    • Special Notes - Modify instructions
    • Order Items - Adjust quantities, prices, or add/remove items
  4. Click Save to update the order
Restriction: Orders can only be edited when in "planned" status. Orders in other statuses must be handled through the order handling interface.


Deleting a Customer Order

To remove a customer order from the system:

  1. Locate the order in the list view
  2. Click the Delete button (trash icon)
  3. Confirm deletion in the dialog box
Important: Orders can only be deleted when in "planned" status. This action cannot be undone and will permanently remove the order record.


Handling Customer Orders

The order handling interface provides specialized actions for managing orders through their lifecycle:

  1. From the orders list, click the Handle action button
  2. The handling page shows order details and available actions
  3. Available actions depend on current status:
    • Release Order (planned → released)
    • Complete Order (cut/started/released → completed)
    • Mark as Delivered (completed → delivered)
    • Cancel Order (planned → cancelled)
    • Pause Order (material released/cut/started → paused)
    • Resume Order (paused → cut)
    • Plan Order (released → planned)
    • Print PDF (all statuses)
Workflow: Each status change is logged in the audit trail with timestamp and user information.

Status Definitions

Status Description Allowed Actions
Planned Order created but not yet processed Edit, Delete, Release
Released Order approved and ready for processing Complete, Pause, Plan Again, Print
Cut Materials have been prepared Complete, Print
Started Production has begun Complete, Print
Completed Production finished Mark Delivered, Print
Delivered Order delivered to customer Invoice, Print
Invoiced Order invoiced Close, Print
Closed Order closed and archived View Only
